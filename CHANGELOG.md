# 更新日志

## 2026-05-12

- 新增节点运营中心：后台可查看节点健康度、TCP 延迟、运行负载、故障原因和节点流量排行，第一版仅做观测诊断，不自动摘除线路。
- node-agent 心跳新增运行指标上报，worker 每分钟生成节点健康检查记录；旧 agent 未上报指标时仍兼容心跳、任务和流量上报。
- 新增用户级限速：管理员可在用户订阅里设置 Mbps 限速，保存为 `speed_limit_bps` 后随 `UPSERT_USER` 下发到出口 node-agent。
- node-agent 基于 Xray Stats 做滑动窗口平均限速，超限时临时摘除用户并在窗口结束后恢复；该能力不改变订阅内容、套餐扣费和流量账本。
- 新增迁移 `017_add_node_operations_center` 与 `018_add_user_speed_limit`，补充节点运营中心接口、前端页面、限速 API、单元测试和 Playwright smoke 覆盖。
- 重新生成并升级 node-agent 镜像包，启用中的出口节点已持续上报运行指标；测试出口节点已避让 443 端口，RayPilot Xray 只监听本次配置的节点端口。
- 更新 README 截图展示，新增节点运营中心截图，并同步 AGENTS.md、开发方案、管理接口、节点代理接口、页面清单和运维手册。
- 订阅公开入口改为 `/sub/{token}`，只下发 Clash/mihomo YAML；前台和后台不再展示 `/clash`、`/base64`、`/plain` 后缀，所有带后缀订阅入口全部下线。
- 订阅配置页新增节点命名模板、地区国旗开关、全局自动测速和自定义订阅分组；管理员可在订阅配置里自定义分组名称并选择节点加入，当前保留 `url-test` 自动测速，不输出 `fallback` 故障转移组。
- 节点新增、编辑和一键部署新增结构化国家/地区字段，并内置全球常用国家/地区库；订阅节点名的图标、地区和缩写来自节点字段，不再根据节点名称关键词猜测。

## 2026-05-11

- 强化 Docker 一键部署闭环：出口节点和中转节点部署时增加镜像包预检、远端预检、容器运行等待、配置就绪、心跳等待、监听进程归属验证和失败诊断清理。
- 出口节点与中转节点一键部署新增目标机本机防火墙放行：UFW 自动执行 `ufw allow <port>/tcp`，firewalld 自动添加 permanent TCP 端口并 reload，避免 Xray/HAProxy 已监听但公网端口被本机防火墙拦截。
- 通过一键部署上线一台脱敏出口节点服务器，创建 TCP 443 与 XHTTP 8443 两条逻辑线路。
- 将新出口 TCP / XHTTP 线路加入所有当前启用套餐使用的节点分组，并确认已有用户同步任务完成。
- 修复新出口 XHTTP 无信号问题：定位为目标机 UFW 未放行 `8443/tcp`，放行后使用 Xray 26.3.27 客户端实测 XHTTP 出站返回 HTTP 204。
- 同步更新 AGENTS.md、开发方案、节点代理部署指南、联调记录和运维排查文档。

## 2026-05-03

- README 联系方式增加 Telegram 群链接，方便 GitHub 访客加入交流群或联系作者。
- 增加 README 展示截图、可复现的脱敏截图脚本、GitHub Actions CI、Dependabot、Issue 模板、PR 模板、贡献指南和安全策略，提升 GitHub 公开项目转化与协作质量。
- 项目公开品牌改为 RayPilot，README 增加 GitHub SEO 副标题、功能关键词、VPN/代理/梯子/机场面板搜索词和 topics 建议。
- 前端页面标题、用户侧和管理后台品牌文案改为 RayPilot。
- 一键部署使用的 node-agent 镜像名、默认容器名和默认节点名改为 RayPilot 命名，并保留旧容器名清理/兼容逻辑。
- 增加出口节点流量同步状态字段，记录最近流量上报时间、成功时间、错误时间、错误次数和最后错误，后台节点列表可直接查看流量上报状态。
- node-agent 出口模式新增本地流量上报队列；采集到 Xray 累计值后先落本地文件，上报失败时保留，恢复后按采集时间顺序补发，降低中心服务短时不可用导致的流量丢失风险。
- `/api/agent/traffic` 支持 `collected_at`，中心按采集时间写入流量快照和账本，补发历史采集结果时保留真实采集时间。
- 断线恢复后的旧批次补发会按 `collected_at` 做幂等保护，早于或等于已有最新快照的批次不再写入快照和账本，避免乱序重复计费。

## 2026-04-30

- 修复订阅 Token 页面、管理员用户订阅弹窗、用户首页和用户订阅页只能使用 Clash 链接的问题，支持在 Clash、Base64、URI 三种订阅格式间切换。
- 修复出口节点流量统计未入账的问题：node-agent 默认启用 `XRAY_API_SERVER=127.0.0.1:10085`，启动时自动补齐 Xray Stats API 配置，并兼容 statsquery 返回的数字或字符串计数。
- 一键部署出口节点时自动传入 Xray Stats API 地址，确保新部署节点能上报用户级流量。
- 修复同一秒内连续流量快照可能取到旧基线的问题，避免 node-agent 重复上报相同累计值时重复计费。
- 修复用户用量 API 中“订阅截止今日”和“全部截止今日”可能漏掉当日最新账本的问题，统一按明日零点作为今日汇总边界。
- 加固 node-agent 镜像构建流程，下载 Xray release 时启用失败检测、重试和 zip 校验。
- 增加流量快照同秒排序、重复累计值不重复计费、用量截止今日汇总回归测试，并在 Playwright smoke 中覆盖订阅格式切换。

注意：已有出口节点需要更新 node-agent 镜像或二进制后，流量统计才会从节点侧开始上报。
