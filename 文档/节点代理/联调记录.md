## 中心地址容灾联调

- node-agent 新增 `CENTER_SERVER_URLS`，支持主/备多个中心入口。
- 生产控制平台主备入口通过未提交环境变量维护，公开文档只保留占位符。
- 一键部署出口节点和中转节点时，后端会把 `CENTER_SERVER_URL` 与 `CENTER_SERVER_URLS` 同时写入 agent 容器。
- 当传入任一已配置入口时，后端会自动补齐其它入口。
- 后台新增“修复中心”入口，调用 `POST /api/admin/nodes/repair-center` 通过 SSH 修复 Docker 或 systemd agent 的中心地址，并记录 `agent_center_repair` 部署日志。
- 修复接口支持按 `node_id`、`node_host_id` 或 `relay_id` 等待新心跳；等待秒数为 0 时只写远端配置不等待。

| 字段 | 值 |
|------|----|

HAProxy 生成的核心配置：

```haproxy
frontend relay_1_24443
  bind *:24443
  mode tcp
  default_backend backend_1

backend backend_1
  mode tcp
  server exit_1 <exit-node-ip>:443 check
```
