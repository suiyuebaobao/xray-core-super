# 节点代理接口文档

本文档描述服务端与 node-agent 之间的通信协议。node-agent 支持三种运行模式：

| 模式 | 绑定表 | Token | 职责 |
|------|--------|-------|------|
| `AGENT_ROLE=exit` | `nodes` | `NODE_TOKEN` | 管理出口节点 xray 用户、Reality 参数和用户级流量 |
| `AGENT_ROLE=multi_exit` | `node_hosts` + `nodes` | `NODE_HOST_TOKEN` | 一台物理服务器一个 agent，管理多个逻辑出口节点和用户级流量 |
| `AGENT_ROLE=relay` | `relays` | `RELAY_TOKEN` | 管理中转节点 HAProxy TCP 转发端口、后端绑定和线路级指标 |

三种模式复用同一个 node-agent 二进制，但接口、Token、任务表和权限范围必须分开。

## 鉴权方式

node-agent 使用独立 Token 鉴权，每个节点有独立的凭证。
服务端仅保存 Token 的 SHA-256 哈希值（`agent_token_hash` 字段），不保存明文。

出口模式请求需在请求体中包含 `node_id` 和明文 `token` 字段。服务端会对 `token` 计算 SHA-256 后做常量时间比对，客户端不要上传 `agent_token_hash`。

中转模式请求需在请求体中包含 `relay_id` 和明文 `token` 字段。`RELAY_TOKEN` 只能访问中转心跳、任务和结果接口，不能访问出口节点用户同步接口，也不能用于订阅下载。

multi_exit 模式请求需在请求体中包含 `node_host_id` 和明文 `token` 字段。服务端验证物理主机 Token 后，还必须校验上报或任务结果中的 `node_id` 属于该 `node_host_id`。

## 接口列表

以下为现有出口模式接口。

### POST /api/agent/heartbeat — 节点心跳上报

node-agent 定时（默认 10 秒）发送心跳，服务端在响应中返回待执行任务列表，并更新节点最后心跳时间。

#### 请求体

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| node_id | number | 是 | 节点 ID |
| version | string | 否 | agent 版本号 |
| token | string | 是 | 节点鉴权 Token 明文 |

#### 响应示例

```json
{
  "success": true,
  "data": {
    "tasks": [
      {
        "id": 1,
        "action": "UPSERT_USER",
        "payload": "{\"action\":\"UPSERT_USER\",\"subscription_id\":1,\"xray_user_key\":\"user@suiyue.local\",\"uuid\":\"xxx\",\"transport\":\"tcp\",\"flow\":\"xtls-rprx-vision\"}",
        "idempotency_key": "task-1-1713916800",
        "lock_token": "abc123..."
      }
    ]
  }
}
```

无任务时返回空数组。

---

### POST /api/agent/task-result — 任务执行结果上报

node-agent 执行完任务后，向服务端上报执行结果。**服务端会校验节点归属和 lock_token。**

#### 请求体

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| node_id | number | 是 | 节点 ID |
| token | string | 是 | 节点鉴权 Token 明文 |
| task_id | number | 是 | 任务 ID |
| lock_token | string | 是 | 任务锁 Token（心跳返回的 lock_token） |
| success | boolean | 是 | 是否执行成功 |
| error | string | 否 | 错误信息（失败时填写） |

#### 响应示例

```json
{
  "success": true,
  "data": null
}
```

---

### POST /api/agent/traffic — 流量数据上报

node-agent 定时上报当前节点各用户的累计流量数据。

#### 请求体

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| node_id | number | 是 | 节点 ID |
| token | string | 是 | 节点鉴权 Token 明文 |
| collected_at | string | 否 | 本轮流量采集时间，RFC3339 格式；补发本地队列时用于保留真实采集时间 |
| items | array | 是 | 流量计数器列表 |

items 数组元素：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| xray_user_key | string | 是 | xray 用户唯一标识 |
| uplink_total | number | 是 | 累计上行流量（字节） |
| downlink_total | number | 是 | 累计下行流量（字节） |

#### 响应示例

```json
{
  "success": true,
  "data": null
}
```

node-agent 出口模式在调用该接口前，会先把每轮采集结果写入本地队列文件。中心不可用或接口失败时，本地队列保留；恢复后按 `collected_at` 从旧到新补发。服务端会把 `collected_at` 用作 `traffic_snapshots.captured_at` 和 `usage_ledgers.recorded_at`，并在 `nodes` 上更新最近流量上报成功或失败状态。

节点状态字段语义：

| 字段 | 说明 |
|------|------|
| `last_traffic_report_at` | 中心最近一次收到该节点流量报告的时间 |
| `last_traffic_success_at` | 中心最近一次成功处理到的节点采集时间 |
| `last_traffic_error_at` | 最近一次处理失败的中心时间 |
| `traffic_error_count` | 连续处理失败次数，成功后清零 |
| `last_traffic_error` | 最近一次处理失败原因 |

如果补发批次的 `collected_at` 早于或等于中心已有最新快照时间，服务端会幂等跳过该旧批次，不再写新快照或账本，避免乱序补发导致重复计费。

## 多出口 IP 模式接口

multi_exit 模式下，中心仍以 `nodes.id` 作为套餐授权、订阅生成和流量账本归属；`node_hosts.id` 只表示物理服务器和唯一 agent 身份。

### POST /api/agent/multi/heartbeat — 物理主机心跳上报

#### 请求体

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| node_host_id | number | 是 | 物理节点服务器 ID |
| version | string | 否 | agent 版本号 |
| token | string | 是 | `NODE_HOST_TOKEN` 明文 |

响应中的任务会额外携带 `node_id`，agent 必须按该逻辑节点执行。

### POST /api/agent/multi/task-result — 多出口节点任务结果

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| node_host_id | number | 是 | 物理节点服务器 ID |
| node_id | number | 是 | 逻辑出口节点 ID，必须属于该 `node_host_id` |
| token | string | 是 | `NODE_HOST_TOKEN` 明文 |
| task_id | number | 是 | 任务 ID |
| lock_token | string | 是 | 任务锁 Token |
| success | boolean | 是 | 是否执行成功 |
| error | string | 否 | 错误信息 |

### POST /api/agent/multi/traffic — 多出口节点流量上报

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| node_host_id | number | 是 | 物理节点服务器 ID |
| token | string | 是 | `NODE_HOST_TOKEN` 明文 |
| reports | array | 是 | 按逻辑节点拆分的流量上报批次 |

`reports` 元素：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| node_id | number | 是 | 逻辑出口节点 ID |
| collected_at | string | 否 | 本轮采集时间，语义与 `/api/agent/traffic` 相同 |
| items | array | 是 | 还原后的中心侧 `xray_user_key` 累计流量 |

multi_exit 模式下，agent 写入 Xray clients 时使用节点内部分隔的统计 email，例如 `node_1__user@suiyue.local`，避免同一用户跨多个逻辑节点时 Xray Stats 累计值混合。上报中心前必须去掉内部前缀，恢复为原始 `xray_user_key`，并通过 `node_id` 表示流量归属。

## 任务类型

| 任务类型 | 说明 | payload 内容 |
|----------|------|--------------|
| UPSERT_USER | 开通/续费用户 | JSON：包含 action、subscription_id、xray_user_key、uuid、transport、flow；XHTTP 节点 `transport=xhttp` 且不包含 flow |
| DISABLE_USER | 禁用用户 | JSON：包含 action、subscription_id、xray_user_key、uuid |

## 中转模式接口

### POST /api/agent/relay/heartbeat — 中转心跳上报

node-agent relay 模式定时发送心跳，服务端更新中转节点最后心跳时间，并返回待执行的中转配置任务。

#### 请求体

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| relay_id | number | 是 | 中转节点 ID |
| version | string | 否 | agent 版本号 |
| token | string | 是 | `RELAY_TOKEN` 明文 |

#### 响应任务

任务来自 `relay_config_tasks`，第一版只下发完整配置刷新任务：

| 任务类型 | 说明 |
|----------|------|
| RELOAD_CONFIG | 重新渲染 HAProxy 配置，校验通过后 reload |

`RELOAD_CONFIG` payload 示例：

```json
{
  "action": "RELOAD_CONFIG",
  "forwarder_type": "haproxy",
  "backends": [
    {
      "id": 10,
      "name": "香港中转",
      "exit_node_id": 1,
      "listen_port": 24443,
      "target_host": "10.0.0.2",
      "target_port": 443,
      "is_enabled": true
    }
  ]
}
```

### POST /api/agent/relay/task-result — 中转任务结果上报

node-agent relay 模式完成配置渲染、配置校验和 reload 后，上报任务结果。服务端必须校验 `relay_id`、`RELAY_TOKEN` 和 `lock_token`。

### POST /api/agent/relay/traffic — 中转线路级指标上报

node-agent relay 模式通过 HAProxy stats socket 读取 `show stat` 后上报中转线路级指标，用于运维分析，不作为用户套餐扣量依据。默认 socket 为 `/tmp/haproxy.sock`，可通过 `HAPROXY_STATS_SOCKET_PATH` 覆盖。

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| relay_id | number | 是 | 中转节点 ID |
| token | string | 是 | `RELAY_TOKEN` 明文 |
| items | array | 是 | 线路级指标列表 |

items 数组元素：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| relay_backend_id | number | 否 | 中转后端绑定 ID |
| listen_port | number | 是 | 中转监听端口 |
| bytes_in_total | number | 是 | 端口累计入站字节 |
| bytes_out_total | number | 是 | 端口累计出站字节 |

用户套餐扣量仍由出口模式 `/api/agent/traffic` 按用户 UUID 上报的数据决定。

## 重试策略

- 任务最大重试次数由 `TASK_RETRY_LIMIT` 环境变量控制（默认 10）
- 超过重试次数仍未完成的任务，标记为 FAILED，等待人工介入
- 任务锁有效期由 `TASK_LOCK_TTL` 环境变量控制（默认 120 秒）
- 超过锁有效期未完成的任务，可被后续扫描重新接管
