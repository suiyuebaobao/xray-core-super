# 2026-05-07 日报

## 今日事项

- 确认新需求落地口径：一个套餐内同时支持普通流量与家宽流量两个独立流量池。
- 约束双流量池计费规则：节点声明自身流量池，用户连接哪类节点就扣哪类流量，两个流量池不能混扣。
- 更新三份规则文件与开发方案，补充双流量池字段、节点流量池归属、按池订阅过滤和按池禁用要求。

## 已确定的实现原则

- `plans.traffic_limit` / `user_subscriptions.traffic_limit` / `user_subscriptions.used_traffic` 继续作为普通流量兼容字段。
- 新增家宽流量独立字段，避免一次性推翻现有数据结构和前端接口。
- `nodes.traffic_pool` 固定使用 `normal` 或 `residential`，流量上报按节点池扣量。
- 订阅超额不再简单挂起整个订阅，而是按流量池禁用对应节点线路。
