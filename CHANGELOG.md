# 更新日志

## 2026-05-03

- 增加出口节点流量同步状态字段，记录最近流量上报时间、成功时间、错误时间、错误次数和最后错误，后台节点列表可直接查看流量上报状态。
- node-agent 出口模式新增本地流量上报队列；采集到 Xray 累计值后先落本地文件，上报失败时保留，恢复后按采集时间顺序补发，降低中心服务短时不可用导致的流量丢失风险。
- `/api/agent/traffic` 支持 `collected_at`，中心按采集时间写入流量快照和账本，补发历史采集结果时保留真实采集时间。
- 断线恢复后的旧批次补发会按 `collected_at` 做幂等保护，早于或等于已有最新快照的批次不再写入快照和账本，避免乱序重复计费。

## 2026-04-30

- 修复订阅 Token 页面、管理员用户订阅弹窗、用户首页和用户订阅页只能使用 Clash 链接的问题，支持在 Clash、Base64、URI 三种订阅格式间切换。
- 修复出口节点流量统计未入账的问题：node-agent 默认启用 `XRAY_API_SERVER=127.0.0.1:10085`，启动时自动补齐 Xray Stats API 配置，并兼容 statsquery 返回的数字或字符串计数。
- 一键部署出口节点时自动传入 Xray Stats API 地址，确保新部署节点能上报用户级流量。
- 修复同一秒内连续流量快照可能取到旧基线的问题，避免 node-agent 重复上报相同累计值时重复计费。
- 修复用户用量 API 中“订阅截止今日”和“全部截止今日”可能漏掉当日最新账本的问题，统一按明日零点作为今日汇总边界。
- 加固 node-agent 镜像构建流程，下载 Xray release 时启用失败检测、重试和 zip 校验。
- 增加流量快照同秒排序、重复累计值不重复计费、用量截止今日汇总回归测试，并在 Playwright smoke 中覆盖订阅格式切换。

注意：已有出口节点需要更新 node-agent 镜像或二进制后，流量统计才会从节点侧开始上报。
