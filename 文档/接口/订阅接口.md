# 接口文档 - 订阅接口

订阅接口使用用户级订阅 Token。每个用户只有一个 Token，直连线路和中转线路都在同一份订阅内容中返回，不为中转单独生成 Token。

## 线路生成规则

系统以出口节点作为授权核心：

- 直连线路：`server/port` 使用出口节点 `nodes.host:nodes.port`
- 中转线路：`server/port` 使用中转节点 `relays.host:relay_backends.listen_port`
- 两类线路的 `uuid/sni/pbk/sid/fingerprint/flow/transport/xhttp-*` 都使用对应出口节点参数；XHTTP 线路和 SOCKS5 家宽线路不下发 `flow`
- Clash/mihomo YAML 的 `udp` 字段来自 `nodes.udp_enabled`；SOCKS5 家宽线路默认 `udp: false`，用于避免 QUIC/HTTP3 经二次转发导致的卡顿或风控
- 如果一台服务器同时启用 TCP 与 XHTTP，中心会把它们保存为两条逻辑出口节点；订阅按两条独立线路输出，端口和传输参数分别来自对应 `nodes` 记录
- 中转节点只做 TCP 透传，不终止 Reality，不保存用户 UUID，不参与流量计费
- 用户连接中转不需要中转专用用户 Token，客户端携带的 VLESS UUID 会透传到出口节点，由出口节点完成用户鉴权和用户级流量统计

如果需要隐藏出口 IP，应配置出口节点为仅中转下发，并在出口节点防火墙只放行中转服务器 IP。只从订阅中去掉直连线路不足以完成隐藏。

## 订阅下载

**GET /sub/:token**

返回 Clash/mihomo YAML 格式订阅配置。`/sub/:token/clash`、`/sub/:token/base64` 与 `/sub/:token/plain` 等带后缀入口已全部下线，不能继续用于订阅下载。

### 鉴权

通过 URL 路径中的 token 参数鉴权，不需要 JWT。

### 响应头

```
Content-Type: text/yaml; charset=utf-8
Content-Disposition: attachment; filename="RayPilot"; filename*=UTF-8''RayPilot
ETag: "sub-1-2-1713916800"
profile-title: base64:UmF5UGlsb3Q=
profile-update-interval: 24
subscription-userinfo: upload=0; download=5368709120; total=107374182400; expire=1716508800
```

订阅名称、下载文件名、Clash/mihomo 规则、节点命名模板、订阅分组、自动测速、`profile-update-interval`、`profile-web-page-url` 和 `subscription-userinfo` 由后台 `/admin/subscription-settings` 维护，配置存储在 `site_settings.setting_key=subscription_config`。未保存后台配置时，订阅名称回退到服务端环境变量 `SUBSCRIPTION_PROFILE_NAME`，默认 `RayPilot`。Clash/mihomo 格式的下载文件名与 `profile-title` 使用纯订阅名称，不追加 `.yaml`。

节点国旗来自 `nodes.region_flag`，国家/地区名称来自 `nodes.region_name`，地区缩写来自 `nodes.region_code`，由后台节点新增、编辑或一键部署时从内置全球常用国家/地区库选择。订阅不会根据节点名称猜测国家/地区；未选择地区时不强行追加国旗。节点命名模板支持 `{{flag}}`、`{{name}}`、`{{region}}`、`{{code}}`、`{{index}}`、`{{transport}}`、`{{pool}}`。

`proxy-groups` 由后台订阅配置手动维护。管理员可以自定义分组名称，并选择节点加入分组；当前支持 `select` 和 `url-test`，自动测速组使用 `health_check_url` 与 `url_test_interval`。当前版本不输出 `fallback` 故障转移组。

`subscription-userinfo` 采用常见订阅客户端响应头格式：

- `upload=0`：当前系统没有按订阅头拆分上传/下载展示，上传固定为 0。
- `download`：普通流量池与家宽流量池合计已用字节数。
- `total`：普通流量池与家宽流量池合计总量字节数；普通流量池不限量时不计入总量。
- `expire`：当前有效订阅到期时间的 Unix 秒级时间戳。

### 错误响应

| 错误码 | 说明 |
|--------|------|
| 40004 | 订阅已过期或不可用 |
| 40007 | 当前套餐没有可用节点，通常是套餐未绑定节点分组、节点已停用，或该流量池已无可用额度 |
| 40001 | 请求参数错误 |

## 鉴权规则

订阅 token 必须满足：
- token 存在且未吊销
- token 所属用户状态为 active
- token 所属用户当前订阅状态为 ACTIVE
- 订阅未过期
- 流量未超限
- 请求路径为 `/sub/{token}`，不带任何格式后缀

每个用户只保留一个订阅 token。管理员重置 token 时会更新同一条记录的 token 值。

## 中转线路验收

订阅中出现中转线路时，至少确认：

- 订阅 URL 返回 200，且中转 YAML 代理项的 `server/port` 指向中转节点
- 中转监听端口 TCP 可达
- 出口节点 Xray 端口允许中转节点 IP 访问
- 中转 YAML 代理项的 `sni/pbk/sid` 与出口节点 Xray Reality 配置一致
- 用户 UUID 已由 node-agent 写入出口节点 Xray `clients`

## 缓存策略

- 服务端生成 ETag 响应头
- 客户端可发送 `If-None-Match` 请求头
- 匹配则返回 304 Not Modified
- 建议客户端缓存 30-60 秒
