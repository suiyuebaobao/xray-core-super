# 前端架构说明

## 技术栈

- **Vue 3** — 组合式 API
- **Vite 6** — 开发服务器和构建工具
- **Element Plus** — UI 组件库
- **Alova** — HTTP 请求封装（内置缓存、分页）
- **Pinia** — 状态管理
- **Vue Router 4** — 路由管理

## 鉴权与交互

### 双 Token 机制

1. **登录**：服务端返回 Access Token（JSON body）+ Refresh Token（HttpOnly Cookie）
2. **请求**：Access Token 存储在内存中，Alova 拦截器自动注入 `Authorization: Bearer` 头
3. **刷新**：Access Token 过期时，前端通过统一 Alova 实例调用 `/api/auth/refresh`（浏览器自动携带 Cookie）
4. **登出**：调用 `/api/auth/logout`，服务端删除 Refresh Token 记录

### 路由守卫

| Meta | 行为 | 适用路由 |
|------|------|----------|
| `requiresAuth` | 未登录跳转 `/login` | `/`, `/subscription`, `/orders` 等 |
| `requiresAdmin` | 非管理员跳转 `/admin/login` | `/admin/*` |
| `guest` | 已登录跳转 `/` | `/login`, `/register`, `/admin/login` |

### CSRF 防护

- Refresh Token 使用 HttpOnly Cookie，前端 JS 不可读取
- SameSite Cookie 属性由服务端 Set-Cookie 时设置
- 写操作请求头需带 `X-CSRF-Token: suiyue-web`，由 `frontend/src/api/request.js` 统一注入

### API 适配层

页面不直接拼接 `/api/*` URL；统一通过 `frontend/src/api/index.js` 中的 `authApi`、`userApi`、`planApi`、`redeemApi`、`adminApi` 调用。分页参数在适配层统一收口，页面只传 `{ page, size, keyword }` 等结构化参数。

## 页面清单与路径

| 页面 | 路径 | 说明 |
|------|------|------|
| 用户登录 | `/login` | 用户名 + 密码 |
| 用户注册 | `/register` | 用户名 + 密码 + 邮箱（可选） |
| 用户首页 | `/` | 订阅概览、快捷入口 |
| 我的订阅 | `/subscription` | 订阅信息、用量记录、三种格式链接 |
| 订单列表 | `/orders` | 用户订单历史（v1 骨架） |
| 套餐列表 | `/plans` | 上架套餐展示 |
| 兑换码 | `/redeem` | 提交兑换码 |
| 个人中心 | `/profile` | 修改邮箱和密码 |
| 管理登录 | `/admin/login` | 管理员登录 |
| 管理首页 | `/admin` | 统计概览 |
| 套餐管理 | `/admin/plans` | CRUD、列表展示基础套餐标识和绑定节点分组，基础套餐不可删除，支持多选批量删除普通套餐 |
| 节点组管理 | `/admin/node-groups` | CRUD、列表展示组内出口节点、直连/中转线路类型和组内节点多选绑定，支持多选批量删除分组 |
| 节点管理 | `/admin/nodes` | 出口节点 CRUD、一键部署、直连/中转下发策略，支持多选批量删除 |
| 中转节点管理 | `/admin/relays` | 中转入口、监听端口、出口节点绑定、node-agent relay 模式状态；后端绑定通过选择出口节点自动带出目标地址，只保留端口可编辑，支持多选批量删除 |
| 用户管理 | `/admin/users` | 新增、列表直接展示套餐和剩余流量、禁用/启用、重置密码、订阅管理、用量记录，支持多选批量删除非当前登录用户 |
| 订单管理 | `/admin/orders` | 订单列表（v1 骨架） |
| 兑换码管理 | `/admin/redeem-codes` | 批量生成、列表；生成时按套餐名称选择，列表展示套餐名称 |
| 订阅 Token 管理 | `/admin/subscription-tokens` | 按用户查看唯一 Token、订阅链接、Token 状态、最后使用、Token 过期时间、套餐名称、套餐到期和流量用量 |

## 端到端测试

前端已接入 Playwright + Chromium，用于验证真实浏览器中的登录、路由渲染、页面核心文案和接口请求状态。

```bash
cd frontend
E2E_ADMIN_USERNAME=admin E2E_ADMIN_PASSWORD=your-password npm run test:e2e -- --workers=1
```

默认测试地址为 `http://127.0.0.1:7000`，可通过 `E2E_BASE_URL` 覆盖：

```bash
E2E_BASE_URL=http://156.238.231.216 E2E_ADMIN_USERNAME=admin E2E_ADMIN_PASSWORD=your-password npm run test:e2e -- --workers=1
```

需要把真实 multi_exit 流量上报纳入验收时，额外设置物理节点和逻辑节点环境变量：

```bash
E2E_REQUIRE_MULTI_TRAFFIC=1 \
E2E_MULTI_NODE_HOST_ID=<node-host-id> \
E2E_MULTI_TRAFFIC_NODE_IDS=<node-id-a,node-id-b> \
E2E_MULTI_NODE_HOST_TOKEN=<node-host-token> \
E2E_ADMIN_USERNAME=admin \
E2E_ADMIN_PASSWORD=your-password \
npm run test:e2e -- --workers=1
```

完整套件建议使用 `--workers=1`。当前后台登录态使用同一管理员账号，串行执行可以避免多个测试文件同时刷新登录态导致互相下线；真实多用户行为由测试内部的多个 browser context 和 API actor 模拟。

测试覆盖：

- 管理后台登录和 Dashboard
- `/admin`、套餐、节点分组、出口节点、中转节点、用户、订单、兑换码、订阅 Token 页面
- 用户首页、我的订阅、订单、套餐、兑换码、个人资料页面
- 浏览器控制台错误、请求失败和 5xx 响应检查
- 管理后台 CRUD API 闭环：套餐、节点分组、出口节点、中转节点、relay 后端绑定、用户状态、重置密码、订阅开通/调整、用户用量查询、订阅 Token 重置/撤销、订单创建/列表、兑换码生成/兑换
- 管理后台真实 UI 主路径：节点新增/编辑、一键部署多 IP 开关、节点分组绑定、套餐绑定、relay 后端绑定、用户订阅、用量弹窗、订单、兑换码、订阅 Token 格式切换
- 多用户真实路径：两个独立用户注册登录、套餐浏览、下单、兑换码开通、订阅页、Clash/Base64/URI 下载、个人资料更新和用户数据隔离
- 订阅下载响应使用 `SUBSCRIPTION_PROFILE_NAME` 生成 `Content-Disposition` 文件名，默认 `RayPilot.yaml` / `RayPilot.txt`，客户端导入后不再默认显示 `config.yaml`
- 流量数据链路：node-agent 心跳领取任务、任务结果上报、单节点流量基线/增量/乱序旧批次/计数器归零/恢复增长、真实 multi_exit 多逻辑节点上报、超额挂起和用量页面展示

E2E 测试会创建 `e2e-crud-*`、`e2e-ops-*`、`real-user-flow-*`、`traffic-flow-*` 前缀的临时数据。支持删除接口的套餐、节点、节点分组、中转节点会在测试结束时清理；订单、兑换码等历史型数据会保留业务历史，测试会撤销 Token、禁用或删除测试用户、下架测试套餐并解绑节点分组。
