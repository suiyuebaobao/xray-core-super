# 接口文档 - 订阅接口

订阅接口使用用户级订阅 Token。每个用户只有一个 Token，直连线路和中转线路都在同一份订阅内容中返回，不为中转单独生成 Token。

## 线路生成规则

系统以出口节点作为授权核心：

- 直连线路：`server/port` 使用出口节点 `nodes.host:nodes.port`
- 中转线路：`server/port` 使用中转节点 `relays.host:relay_backends.listen_port`
- 两类线路的 `uuid/sni/pbk/sid/fingerprint/flow` 都使用对应出口节点参数
- 中转节点只做 TCP 透传，不终止 Reality，不保存用户 UUID，不参与流量计费
- 用户连接中转不需要中转专用用户 Token，客户端携带的 VLESS UUID 会透传到出口节点，由出口节点完成用户鉴权和用户级流量统计

如果需要隐藏出口 IP，应配置出口节点为仅中转下发，并在出口节点防火墙只放行中转服务器 IP。只从订阅中去掉直连线路不足以完成隐藏。

## Clash 订阅下载

**GET /sub/:token/clash**

返回 Clash/mihomo YAML 格式订阅配置。

### 鉴权

通过 URL 路径中的 token 参数鉴权，不需要 JWT。

### 响应头

```
Content-Type: text/yaml; charset=utf-8
Content-Disposition: attachment; filename="RayPilot.yaml"; filename*=UTF-8''RayPilot.yaml
ETag: "sub-1-2-1713916800"
```

文件名基础名称由服务端环境变量 `SUBSCRIPTION_PROFILE_NAME` 控制，默认 `RayPilot`。客户端导入后显示的订阅名称取决于客户端实现，通常会优先使用该文件名。

### 响应示例

```yaml
mixed-port: 7890
allow-lan: false
mode: Rule
log-level: info
dns:
  enable: true
  enhanced-mode: fake-ip
  nameserver:
    - 223.5.5.5
    - 119.29.29.29
proxies:
  - name: HK-01 直连
    type: vless
    server: hk.example.com
    port: 443
    uuid: user-uuid
    network: tcp
    udp: true
    tls: true
    servername: www.microsoft.com
    flow: xtls-rprx-vision
    client-fingerprint: chrome
    reality-opts:
      public-key: pubkey
      short-id: sid
  - name: 香港中转-01 -> HK-01
    type: vless
    server: relay-hk.example.com
    port: 24443
    uuid: user-uuid
    network: tcp
    udp: true
    tls: true
    servername: www.microsoft.com
    flow: xtls-rprx-vision
    client-fingerprint: chrome
    reality-opts:
      public-key: pubkey
      short-id: sid
proxy-groups:
  - name: PROXY
    type: select
    proxies:
      - HK-01 直连
      - 香港中转-01 -> HK-01
      - DIRECT
rules:
  - GEOIP,CN,DIRECT
  - MATCH,PROXY
```

### 错误响应

| 错误码 | 说明 |
|--------|------|
| 40004 | 订阅已过期或不可用 |
| 40001 | 请求参数错误 |

## Base64 聚合订阅

**GET /sub/:token/base64**

返回 Base64 编码的 VLESS URI 聚合订阅。编码前内容与 `/sub/:token/plain` 一致，为每行一个 URI，不是 Clash YAML。直连和中转线路都会作为独立 URI 出现在编码前内容中。

### 响应头

```
Content-Type: text/plain; charset=utf-8
Content-Disposition: attachment; filename="RayPilot.txt"; filename*=UTF-8''RayPilot.txt
ETag: "sub-1-2-1713916800"
```

### 响应示例

```
dmxlc3M6Ly91dWlkQGhrLmV4YW1wbGUuY29tOjQ0Mz9lbmNyeXB0aW9uPW5vbmUm...
```

## 纯文本 URI 下载

**GET /sub/:token/plain**

返回纯文本 URI 分享链接（每行一个 VLESS 链接）。

### 响应头

```
Content-Type: text/plain; charset=utf-8
Content-Disposition: attachment; filename="RayPilot.txt"; filename*=UTF-8''RayPilot.txt
ETag: "sub-1-2-1713916800"
```

### 响应示例

```
vless://uuid@hk.example.com:443?encryption=none&flow=xtls-rprx-vision&security=reality&sni=www.microsoft.com&pbk=pubkey&sid=sid&fp=chrome&type=tcp#HK-01%20直连
vless://uuid@relay-hk.example.com:24443?encryption=none&flow=xtls-rprx-vision&security=reality&sni=www.microsoft.com&pbk=pubkey&sid=sid&fp=chrome&type=tcp#香港中转-01%20-%3E%20HK-01
```

## 鉴权规则

订阅 token 必须满足：
- token 存在且未吊销
- token 所属用户状态为 active
- token 所属用户当前订阅状态为 ACTIVE
- 订阅未过期
- 流量未超限

每个用户只保留一个订阅 token。管理员重置 token 时会更新同一条记录的 token 值。

## 中转线路验收

订阅中出现中转线路时，至少确认：

- 订阅 URL 返回 200，且中转 URI 的 `server/port` 指向中转节点
- 中转监听端口 TCP 可达
- 出口节点 Xray 端口允许中转节点 IP 访问
- 中转 URI 的 `sni/pbk/sid` 与出口节点 Xray Reality 配置一致
- 用户 UUID 已由 node-agent 写入出口节点 Xray `clients`

## 缓存策略

- 服务端生成 ETag 响应头
- 客户端可发送 `If-None-Match` 请求头
- 匹配则返回 304 Not Modified
- 建议客户端缓存 30-60 秒
