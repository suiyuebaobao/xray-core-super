# 生产部署指南

本文档描述 suiyue（岁月）系统的生产环境部署步骤。

## 1. 环境要求

- Linux 服务器（推荐 Ubuntu 22.04+ / Debian 12+）
- Docker 24.0+ 和 Docker Compose。本项目默认使用 `docker-compose` 命令，Makefile 可通过 `COMPOSE="docker compose"` 覆盖。
- 至少 2GB 内存、20GB 磁盘
- 已备案域名（用于 HTTPS）
- SSL 证书（Let's Encrypt 或自有证书）

## 2. 部署架构

```
互联网 → Nginx（HTTPS）→ 前端静态资源 + API 反向代理
                                    ↓
                              Go API（:3000）
                                    ↓
                              MySQL 8.4（内网）
```

- 前端静态文件和后端 API 通过 Nginx 统一暴露在 443 端口
- MySQL 不暴露到宿主机，仅容器内部访问
- Worker 进程独立运行，不接收外部请求

## 3. 部署步骤

### 3.1 安装 Docker

```bash
curl -fsSL https://get.docker.com | sh
sudo usermod -aG docker $USER
```

### 3.2 克隆项目

```bash
git clone <repo-url> /opt/suiyue
cd /opt/suiyue
```

### 3.3 配置环境变量

```bash
cp .env.example .env
```

编辑 `.env`，修改以下关键变量：

| 变量 | 说明 | 示例 |
|------|------|------|
| `DATABASE_URL` | MySQL 连接串 | `root:password@tcp(mysql:3306)/suiyue?charset=utf8mb4&parseTime=True` |
| `MIGRATE_DATABASE_URL` | golang-migrate 连接串 | `mysql://root:password@tcp(mysql:3306)/suiyue?charset=utf8mb4&parseTime=true` |
| `NGINX_HTTP_PORT` | Nginx 宿主机 HTTP 端口 | `80` |
| `JWT_SECRET` | JWT 签名密钥（必须随机生成） | `openssl rand -hex 32` |
| `AGENT_AUTH_MODE` | 节点鉴权模式 | `token` |
| `XRAY_USER_KEY_DOMAIN` | xray 用户密钥域名 | `suiyue.local` |
| `SUBSCRIPTION_PROFILE_NAME` | 客户端导入订阅时使用的配置文件名基础名称 | `RayPilot` |

**安全提醒**：
- `JWT_SECRET` 必须使用强随机字符串，否则启动时 panic
- 数据库密码不应使用默认值

### 3.4 数据库迁移

```bash
make migrate
```

### 3.5 创建初始管理员账号

```bash
make seed
```

默认管理员账号：`admin` / `admin123456`，首次登录后请立即修改密码。

### 3.6 构建并启动

```bash
make docker   # 构建服务镜像
make up       # 启动服务
```

### 3.7 验证服务

```bash
# 检查容器状态
docker-compose ps

# 测试 API
curl http://localhost:3000/api/plans

# 查看日志
docker-compose logs -f api
docker-compose logs -f worker
```

## 4. Nginx 反代配置

### 4.1 安装 Nginx

```bash
sudo apt install nginx certbot python3-certbot-nginx
```

### 4.2 配置文件

`/etc/nginx/sites-available/suiyue.conf`：

```nginx
server {
    listen 80;
    server_name your-domain.com;
    # Let's Encrypt 验证用
    location /.well-known/acme-challenge/ {
        root /var/www/certbot;
    }
}
```

```bash
sudo ln -s /etc/nginx/sites-available/suiyue.conf /etc/nginx/sites-enabled/
sudo certbot --nginx -d your-domain.com
```

Certbot 自动续期后，Nginx 配置会自动更新为 HTTPS。

### 4.3 HTTPS 最终配置

```nginx
server {
    listen 443 ssl http2;
    server_name your-domain.com;

    # SSL 证书（由 Certbot 自动管理）
    ssl_certificate /etc/letsencrypt/live/your-domain.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/your-domain.com/privkey.pem;

    # 安全头
    add_header Strict-Transport-Security "max-age=63072000" always;
    add_header X-Frame-Options "SAMEORIGIN" always;
    add_header X-Content-Type-Options "nosniff" always;

    # 前端静态资源
    location / {
        root /opt/suiyue/web/static;
        try_files $uri $uri/ /index.html;
    }

    # API 反向代理
    location /api/ {
        proxy_pass http://127.0.0.1:3000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    # 订阅接口
    location /sub/ {
        proxy_pass http://127.0.0.1:3000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
```

## 5. Docker Compose 服务说明

| 服务 | 职责 | 重启策略 |
|------|------|----------|
| `api` | 后端 API + 订阅接口 | always |
| `worker` | 定时任务（流量同步、过期扫描等） | always |
| `mysql` | MySQL 8.4 数据库 | always |

### 5.1 端口映射

- API：3000 → 3000（仅 Nginx 反代时暴露，生产环境可通过 Nginx 代理）
- MySQL：不暴露到宿主机

## 6. 宿主机 systemd 部署方式

若生产环境采用宿主机 Nginx 直接反代到宿主机二进制，而不是通过 Compose 暴露 API，需要用 systemd 托管 API 和 Worker，避免 SSH 断开或终端退出导致后端进程被回收。

仓库提供服务文件：

```bash
deploy/systemd/suiyue-api.service
deploy/systemd/suiyue-worker.service
```

生产环境变量放在 `/etc/suiyue/suiyue.env`，不要提交到仓库。该文件至少包含：

```bash
DATABASE_URL='user:password@tcp(127.0.0.1:3306)/suiyue?charset=utf8mb4&parseTime=True&loc=Local'
MIGRATE_DATABASE_URL='mysql://user:password@tcp(127.0.0.1:3306)/suiyue?charset=utf8mb4&parseTime=True&loc=Local'
APP_PORT=3000
JWT_SECRET='replace-with-random-secret'
```

安装并启动：

```bash
install -m 600 /path/to/suiyue.env /etc/suiyue/suiyue.env
cp deploy/systemd/suiyue-api.service /etc/systemd/system/suiyue-api.service
cp deploy/systemd/suiyue-worker.service /etc/systemd/system/suiyue-worker.service
systemctl daemon-reload
systemctl enable --now suiyue-api suiyue-worker
systemctl status suiyue-api suiyue-worker
```

验证 Nginx upstream：

```bash
curl -i http://127.0.0.1:3000/health
curl -i http://127.0.0.1:7001/health
curl -i http://your-domain-or-ip/api/plans
```

## 7. 数据持久化

MySQL 数据存储在 Docker Volume `mysqldata`，映射到 `/var/lib/mysql`。

备份：

```bash
docker-compose exec mysql mysqldump -u root -p suiyue > backup_$(date +%Y%m%d).sql
```

恢复：

```bash
docker-compose exec -T mysql mysql -u root -p suiyue < backup_20260425.sql
```

```bash
make docker   # 重新构建
make down     # 停止旧容器
make up       # 启动新容器
make migrate  # 执行新的迁移
```

```bash
# 回退到上一版本
git checkout HEAD~1
make docker
make down
make up
# 如有数据库迁移，需手动回滚数据
```

## 11. 生产验收边界

上线前至少完成以下验证：

```bash
go test ./...
docker-compose config --services
cd frontend
npm run build
E2E_ADMIN_USERNAME=<admin> E2E_ADMIN_PASSWORD=<password> npm run test:e2e -- --workers=1
```

如果本次包含多出口 IP 或流量账本变更，Playwright 验收还应带上真实 multi_exit 环境变量：

```bash
E2E_REQUIRE_MULTI_TRAFFIC=1 \
E2E_MULTI_NODE_HOST_ID=<node-host-id> \
E2E_MULTI_TRAFFIC_NODE_IDS=<node-id-a,node-id-b> \
E2E_MULTI_NODE_HOST_TOKEN=<node-host-token> \
E2E_ADMIN_USERNAME=<admin> \
E2E_ADMIN_PASSWORD=<password> \
npm run test:e2e -- --workers=1
```

公网入口还需要确认：

```bash
curl -i http://your-domain-or-ip/api/plans
curl -i http://your-domain-or-ip/sub/<token>/clash
```

如果本次包含节点、订阅或中转变更，不能只看订阅能导入。必须用真实客户端或 Xray 本地代理验证至少一条直连线路和一条中转线路可以出站；中转线路还要检查中转端口可达、`relay_config_tasks` 最新任务成功、`relay_traffic_snapshots` 出现非零流量。

当前第一版中转能力已通过核心路径验证，但以下项目仍属于生产前风险项：

- 长时间稳定性观察和并发压测。
- 支付回调验签、幂等、对账和异常补偿全链路。
- 隐藏出口 IP 模式下的出口节点防火墙封锁验证。
- 管理后台主路径 CRUD 已由 Playwright 覆盖；破坏性操作的复杂组合、长时间稳定性和并发压测仍需单独验收。
