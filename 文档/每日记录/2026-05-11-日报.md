## XHTTP 故障定位

- 现象：已有 XHTTP 正常，新增 XHTTP 线路在客户端无信号。
- 中心记录检查：新增 XHTTP 节点的 `transport=xhttp`、端口 `8443`、Reality 公钥、SNI、XHTTP path 均正确。
- 订阅检查：Clash 订阅中已下发 `network: xhttp`、`xhttp-opts.path=/raypilot`、`server=<exit-node-ip>`、`port=8443`。
- 远端检查：Xray 已监听 `<exit-node-ip>:8443`，容器运行正常。
- 外部连通检查：部署后 `<exit-node-ip>:443` 可达，但 `<exit-node-ip>:8443` 超时。
- 根因：目标机 UFW 处于 active 状态，已有 `443/tcp` 放行，但缺少 `8443/tcp`。
- 修复：执行 `ufw allow 8443/tcp` 后，外部 `8443` 可达，Xray 客户端通过 XHTTP 出站访问 `https://www.gstatic.com/generate_204` 返回 HTTP 204。

## 代码修复

- `NodeDeployService` 新增 `ensureFirewallPortsOpen`：
  - UFW active 时自动 `ufw allow <port>/tcp`
  - firewalld active 时自动 `firewall-cmd --permanent --add-port=<port>/tcp` 并 reload
  - 未启用 UFW/firewalld 时跳过，不阻断部署
- 出口节点一键部署在端口占用检查后自动放行本次节点端口。
- 多传输 / multi_exit 一键部署会按所有逻辑节点端口去重后放行，例如 `443` 与 `8443`。
- 中转节点一键部署会自动放行中转监听端口。
- 新增 `deployPortsFromEndpoints` 单元测试，覆盖端口去重和排序。

## Agent 通信口径

- node-agent 与中心服务是单向主动通信：agent 主动 `POST` 到控制中心，不需要中心主动访问 agent。
- 出口 multi_exit 使用：
  - `POST /api/agent/multi/heartbeat`
  - `POST /api/agent/multi/task-result`
  - `POST /api/agent/multi/traffic`
- 单出口使用：
  - `POST /api/agent/heartbeat`
  - `POST /api/agent/task-result`
  - `POST /api/agent/traffic`
- 中转 relay 使用：
  - `POST /api/agent/relay/heartbeat`
  - `POST /api/agent/relay/task-result`
  - `POST /api/agent/relay/traffic`
- 鉴权使用节点、物理节点服务器或中转记录对应的明文 Token；中心只保存 SHA-256 hash，不使用管理员 JWT。
